# 2026-05-14 VviHot主题管理与采集优化更新

## 背景

原项目接入识微商情/VviHot后，只能读取平台已有主题。若平台上存在“远大方略、中智咨询、正略钧策、薄云咨询”等咨询类主题，采集结果会偏离通信设备产业情报；若只白名单少数已有主题，采集量又会明显不足。

## 本次修正

1. 新增VviHot主题自动托管能力。
   - 采集前自动读取平台主题列表。
   - 可删除非托管主题，释放平台主题额度。
   - 自动创建 `CT-` 前缀的通信行业组合关键词主题。

2. 新增5个通信行业托管主题。
   - `CT-无线接入与标准`
   - `CT-光通信`
   - `CT-核心网与承载`
   - `CT-通信芯片半导体`
   - `CT-物联网终端`

3. 支持识微商情布尔关键词表达式。
   - 使用 `||` 组合同领域关键词。
   - 使用 `&&` 组合“技术词 + 事件动作词”。
   - 对 `O-RAN`、`ITU-R` 等带连字符关键词加引号，避免被平台解析为排除语法。

4. 新增二次主题过滤。
   - 平台表达式命中偏宽时，项目内部会再次校验内容是否包含对应技术领域核心词。
   - 英文短缩写如 `IoT`、`CPE`、`PA`、`SoC` 使用单词边界匹配，减少误命中。

5. 加固登录流程。
   - 登录后不再立刻调用主题接口。
   - 会轮询确认 `/swsq/topic/pagingByType` 已返回有效JSON，再开始清理/创建主题。

## 新增配置

```env
COLLECTOR__VVIHOT_MANAGE_TOPICS=true
COLLECTOR__VVIHOT_DELETE_UNMANAGED_TOPICS=true
COLLECTOR__VVIHOT_MANAGED_TOPIC_PREFIX=CT-
```

`.env.example` 已同步补充这些配置项。生产使用时，如果不希望自动删除平台原有主题，可将 `COLLECTOR__VVIHOT_DELETE_UNMANAGED_TOPICS=false`。

## 验证结果

单独运行VviHot采集器后，已验证：

- 可自动删除非通信/非托管主题。
- 可自动创建5个 `CT-` 通信行业主题。
- 可采集到约百条平台原始内容。
- 经项目二次过滤后，可保留数十条有效内容。
- 采集结果摘要中会写入 `监测主题: CT-...`，便于后续分析和追踪。

## 仍需继续优化

1. 识微商情平台本身对关键词表达式的命中仍偏宽，后续建议继续细化每个技术领域的排除词。
2. 当前账号主题额度为5个，因此将“核心网”和“承载网”合并为一个主题；如账号额度增加，可拆成6个独立技术领域主题。
3. RSS和Web采集器仍有部分源不可用或依赖Jina API，后续可单独做源质量治理。
