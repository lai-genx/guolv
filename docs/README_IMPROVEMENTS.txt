🔧 配置说明
================================================================================

### 基础配置 (默认已支持)
- HTTP请求超时: 30秒
- 最大重试次数: 3次
- 每个RSS源最多20条
- 每个Web网站最多10条

### 可选高级配置

1. Jina Reader API (用于Web采集)
   ├─ 何时需要: 官网CSS选择器失效时
   ├─ 配置: COLLECTOR__JINA_API_KEY=your_key
   ├─ 获取: https://jina.ai/
   └─ 优点: 提升Web采集的容错能力

2. HTTP/HTTPS代理
   ├─ 何时需要: 访问国际源需要翻墙
   ├─ 配置:
   │   COLLECTOR__HTTP_PROXY=http://proxy:port
   │   COLLECTOR__HTTPS_PROXY=http://proxy:port
   └─ 示例: http://127.0.0.1:7890

3. RSS源自定义
   ├─ 位置: config.py 中的 rss_feeds 列表
   ├─ 添加源: 编辑并重启应用
   └─ 自动跳过: 失败3次的源会被跳过

================================================================================
✨ 特殊功能亮点
================================================================================

1️⃣ RSS源失败自动恢复
   状态          | 行为
   ─────────────────────────────────
   失败1-2次     | 继续尝试
   失败3次       | 本次采集中跳过
   采集成功      | 失败计数重置
   下一次采集    | 重新计数

2️⃣ Web采集多级选择器
   层级1: .news-item (标准选择器)
   层级2: .list-item / article (备选)
   层级3: Jina Reader API (最后手段)

3️⃣ Search采集anti-crawl机制
   ✓ 5个不同User-Agent随机轮转
   ✓ 5个不同Bing选择器应对结构变化
   ✓ 21个搜索词即使某个失败其他继续

4️⃣ 关键词过滤三层策略
   Layer 1: 技术词 (5G, 光通信等)
   Layer 2: 企业词 (华为, Ericsson等)
   Layer 3: 行业词 (通信设备, 基站等)

📞 常见问题
================================================================================

Q1: 为什么采集还是0条?
A1: 可能是网络问题。检查:
   1. 是否有代理需求 (配置HTTP_PROXY)
   2. 源是否真的有内容 (手动访问URL测试)
   3. 关键词过滤是否过严 (检查contains_keywords())

Q2: 怎样添加自己的RSS源?
A2: 在 config.py 中的 rss_feeds 列表添加URL，或编辑 .env:
   COLLECTOR__RSS_FEEDS=url1,url2,url3

Q3: 搜索采集很慢怎么办?
A3: 可能是Bing反爬。尝试:
   1. 减少搜索关键词数量
   2. 增加搜索间隔
   3. 使用代理

Q4: Web采集获取不到数据?
A4: CSS选择器可能已过时。尝试:
   1. 配置Jina Reader API作为fallback
   2. 手动检查官网HTML结构
   3. 修改DEFAULT_SITES中的选择器
